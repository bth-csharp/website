Nu när vi har installerat utvecklingsmiljön så kan vi också verifiera att den är kopplad till Canvas.

## Uppdatera ditt lokala Git-repo

Det är ingen skillnad på de olika brancherna nu och då blir det inget att jämföra mellan brancherna. Om du försöker göra en PR nu får du meddelandet "There isn’t anything to compare.".

Uppdatera din README.md fil och byt ut namnet `csharp-abcd25` till namnet på ditt lokala Git-repo. Uppdatera texten så att det passar ditt repo. Gör add, commit och push på README.md och kolla att namnet är uppdaterat på GitHub.

Så här ser min "README.md" ut:

```bash title="README.md för grmstud"
# csharp-grmstud

Repo för kursen `csharp`. Här lägger jag koden för mina övningar och uppgifter.

## Kmom01

Beskrivning av koden i kmom01.
```

## Gör en PR

Gå till ditt repo på GitHub och klicka på `Pull request` och därefter `New Pull Request`.

import pr1 from '@assets/kursrepo-utvecklingsmiljo/pr1.png';

<Figure 
    src={pr1}
    caption="Ska ny Pull request."
/>

Alla filer du har skapat finns bara på `main` branchen. Med en PR (Pull request) kan vi kopiera den koden till en annan branch. Nu ska du välja vilken branch du ska kopiera den till. Klicka först (1) "base" och välj branchen (2) `bth/submit/test-gitconfig`. Nu kan du se alla de commits och dess data som ska flyttas till `bth/submit/test-gitconfig` branchen. Klicka på (3) `Create pull request`.

import pr2 from '@assets/kursrepo-utvecklingsmiljo/pr2.png';

<Figure 
    src={pr2}
    caption="Välj test-gifconfig branch."
/>


Ge din PR en titel och klicka sen igen på `Create pull request`.

import pr3 from '@assets/kursrepo-utvecklingsmiljo/pr3.png';

<Figure 
    src={pr3}
    caption="Välj en titel och skapa PR."
/>

Nu har din PR skapats. Skrolla ner på sidan så borde du se något som bilden nedanför. Du kanske behöver ladda om sidan för att det ska synas. Efter några sekunder eller någon minut (det varierar hur lång tid det tar) ska det stå `All checks have passed`. Om det inte gör det har något gått fel. Det är svårt att säga vad av allt som kan gått fel. Du kan klicka på den knappen och tas till fel utskriften, du kan försöka tolka det för se vad som gått fel, annars kontakta en lärare för hjälp.

<Figure

Om allt har gått bra ska du automatiskt ha fått ett betyg på uppgiften `Test gitconfig` i Canvas. Gå till Canvas sidan för kursen och kolla om du har fått ett betyg och en kommentar på den uppgiften.

Om du inte har fått betyg `G` och en kommentar har något gått fel, kontakta en lärare.

Avsluta med att gå tillbaka till din pull request och avsluta den. Klicka på `Close pull request`.

## Vanliga fel

### Cannot index array with string "attempt"

import pr_fail_id from '@assets/pr/pr_fail_id.png';

<Figure 
    src={pr_fail_id}
    caption="Lyckat resultat från testerna."
/>

Något är felaktigt med `STUDENT_CANVAS_ID`. När du fixade labbmiljön skulle du skapa variabeln `STUDENT_CANVAS_ID` i ditt repo. Antingen har du gett den fel värde, gett variabeln fel namn eller skapat den som en secret istället för variabel. [Lägg till Canvas student ID i repo](/website/laromaterial/kursrepo/lagg-till-studentid/).

Stäng din PR och skapa en ny igen.

När du ska skapa en PR mot bth/submit/test-gitconfig får du inte skapa den för att `bth/submit/test-gitconfig and main are entirely different commit histories.`.

#### Skapa nytt repo

Du behöver inte göra om alla steg under "Kursrepo", gör bara de som finns länkade här.

import errorImage5 from '@assets/kursrepo-utvecklingsmiljo/end-pr-error5.png';


<Steps>
1. [Skapa kursrepo på GitHub](../../kursrepo/skapa-github-repo)
1.  Innan du kan skapa det lokalt behöver du döpa om det du har lokalt.
    - Öppna terminalen och exekvera följande:
        ```bash title="Döp om ditt repo"
        cd
        cd dbwebb-kurser
        # Change abcd25 to your own student acronym
        REPO=python-abcd25
        ```
        Så här ser det ut för mig. GLÖM INTE att byta ut `abcd25` mot din akronym.
        <Figure 
            src={errorImage5}
            caption="Så här kan det se ut"
        />
1. Skapa nu ditt repo lokalt, jobba igenom artikel "Skapa git repo" men börja där denna länken tar dig, [Initiera ditt git repo med branches
](../../kursrepo/skapa-git-repo/#initiera-ditt-git-repo-med-branches). Du behöver inte göra stegen ovanför det.
1. Jobba igenom [Lägg till Canvas student ID i repo](../../kursrepo/lagg-till-studentid/).
</Steps>

Nu har du ett tomt repo. Istället för att jobba igenom hela utvecklingsmiljön kan du kopiera filerna från ditt gamla repo.

import errorImage6 from '@assets/kursrepo-utvecklingsmiljo/end-pr-error6.png';

#### Kopiera filer till det nya repot

<Steps>
1. I samma terminal som du har gjort övriga kommandon kör följande för att kopiera filerna till ditt nya repo.
    ```bash
    cd
    cd dbwebb-kurser
    rsync -av --exclude='.git' python-old/ $REPO/
    ls $REPO/
    ```
    När du kör `rsync` kommandot kommer du få en lång utskrift efter du har kört sista kommandot. Jag har inte med det på min bild för att kunna visa allt.
    <Figure 
        src={errorImage6}
        caption="Så här kan det se ut"
        />
1. Flytta in i repot och ladda upp filerna till GitHub.
    ```bash
    cd $REPO
    git add .
    git commit -m 'Skapade om repot, kunde inte skapa en PR.'
    git push
    ```
1. I webbläsaren, gå till ditt repo på GitHub, ladda om sidan och kolla att dina filer är där. Om alla filer finns där kan du radera det gamla repot i terminalen.
1. Radera python-old
    ```bash
    cd dbwebb-kurser
    rm -rf python-old
    ```
</Steps>
